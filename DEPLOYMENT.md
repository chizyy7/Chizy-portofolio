# GitHub Pages Deployment Guide

Follow these simple steps to deploy your portfolio website to GitHub Pages:

## Step 1: Access Repository Settings
1. Go to your GitHub repository: https://github.com/chizyy7/Chizy-portofolio
2. Click on the "Settings" tab (near the top of the page)

## Step 2: Navigate to Pages Section
1. In the left sidebar, scroll down and click on "Pages"
2. You'll see the GitHub Pages configuration options

## Step 3: Configure Source
1. Under "Build and deployment" section
2. Find the "Source" dropdown menu
3. Select "Deploy from a branch"
4. Under "Branch", select your main branch (usually `main` or `master`)
5. Select the folder: `/ (root)`
6. Click "Save"

## Step 4: Wait for Deployment
1. GitHub will automatically build and deploy your site
2. This usually takes 1-2 minutes
3. Refresh the page to see the deployment status

## Step 5: Access Your Live Site
Your portfolio will be available at:
```
https://chizyy7.github.io/Chizy-portofolio/
```

## Updating Your Site
Whenever you make changes to your HTML, CSS, or JavaScript files and push them to GitHub, the site will automatically update within a few minutes.

## Troubleshooting
- If the site doesn't appear, check that the branch name is correct
- Ensure `index.html` is in the root directory
- Wait a few minutes for initial deployment
- Check the "Actions" tab to see deployment progress

## Custom Domain (Optional)
If you want to use a custom domain:
1. In the Pages settings, find "Custom domain"
2. Enter your domain name
3. Follow GitHub's instructions for DNS configuration
